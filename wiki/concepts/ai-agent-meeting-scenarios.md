# 聚焦會議 AI Agent 場景統整

[[taiwan-mobile]] 各事業單位於「聚焦會議」與「BU Kickoff」提出之 **AI Agent / 自動化應用場景**
盤點清單 (工作底稿，含多個未填欄位)。反映全公司由下而上蒐集 AI 落地需求的過程，
與 [[data-coe]] 的 [[data-coe-2025-priorities]] 在 AI/人才主題上相互銜接。

## 來源結構 (5 個工作表)
1. **聚焦會議提出場景**：依組別 (1–8)、特定場域、IT 長官、提出人彙整。
2. **通用場景**：跨單位共通需求 (RAG 文件查詢、Email 附件整理、會議後任務整理、本機語意搜尋、檔案歸檔)。
3. **BU Kickoff 1 場景**：依處別，標註通用分類與導入 **Phase 1/2/3**。
4. **BU Kickoff 2 場景**：依處別之細部流程，含提出人與限制 (最詳盡)。
5. **申查處提出場景**：申查處 (稽核/審查) 之專屬場景。

## 場景分類 (反覆出現之類型)
- **自動化報表型 (Phase 2)**：週/月報、業務日報、損益分析、巡檢統計自動產出與寄送。
- **個人 Agent (Phase 3)**：網路維運之派工、告警、5G 異常細胞、CQI 劣化等個人助理。
- **表單/決策判斷型**：PR 單檢核、重分類會科檢核、預算查詢 (含權限控管)。
- **文件/知識查詢 (RAG，Phase 1)**：SOP 查詢、FAQ 知識庫、法規查核、知識整合平台。
- **會議管理**：以 MyBro 呼叫「聽寫大哥」錄音 → AI 摘要與待辦 → 寄送發起人。

## 代表性場景 (與其他頁面連結)
- **請購小幫手**：由 [[procurement-department]] 江處長 (Poyin) 提出，協助開立 RFP、產品規格、PR 欄位、廠商查詢、歷史採購價格、串接簽核 (公文/EP 系統)、交貨跟催、議定價格轉 PO — 直接呼應 [[procurement-management-policy]] 流程。
- **招募全方位 / AI 面試官**：HR 提出履歷初篩、面談安排、AI 面試官 — 與 [[data-coe-2025-priorities]] 之「培訓面試官」一致。
- **財知音 Fin-Tune / BOE 報表 / 預算隨時查**：財務與經營分析之 agent 化。

## 涉及之內部系統
EP 系統、Mybro、ITKM、BOE、ERP、NWIS、AMS、M+、Outlook、ipm、oracle、Amazon QuickSight 等
(顯示場景需跨多系統整合)。

## 與其他頁面的關係
- 倡議組織：[[taiwan-mobile]]，AI/資料能力對口 [[data-coe]]
- 與 [[procurement-management-policy]] (請購小幫手) 及 [[data-coe-2025-priorities]] (面試官/人才) 直接交集
